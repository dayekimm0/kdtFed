*웹브라우저 대표적으로 3개의 언어 및 스타일 시트만을 인식해왔음

1) HTML5 : 마크업 언어
제품 서비스를 만드는데 있어서 전체적인 구조를 세팅하는 역할에 그치는 언어

2) CSS3 : 스타일시트(*언어X)
마크업 언어의 스타일을 생성하고 적용시켜주는 보조 역할

3) JS : "프로그래밍" 언어
* 사용자와의 능동적인 소통을 할 수 있는가? 불가
* 프로그램을 만들어 낼 수 있다는 전제
* 최대한 인간 친화적인 모델 형식 채택
* 명령 // 값 적용 // 찾아와서 실행
* input => process => output
* F/E => 대표적인 언어 // 가장 쉬운 언어

var : 재선언 & 재할당 O - 쓰레기
let : 재선언 X & 재할당 O
const : 재선언 & 재할당 X
프로그래밍 언어는 엄격할수록 안정성이 뛰어남 - 위험요소 감소
